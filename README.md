# Learning When to Think

Train an LLM to adaptively allocate reasoning compute via a 3-action MDP: **continue**, **refine**, or **terminate**.

## Approach

- **Base model**: Qwen2.5-Math-7B-Instruct + LoRA (r=16, alpha=32)
- **Actions**: `<continue>` (keep reasoning), `<refine>` (self-correct), `<terminate>` (stop and answer)
- **Reward**: Adaptive Length Penalty (ALP) — penalizes length proportional to group solve rate
- **Training**: GRPO with DeGRPO weighting (control tokens get higher gradient weight)
- **Evaluation**: MATH-500

## Project Structure

```
src/
  data/math.py          # MATH-500 loading, answer extraction, grading
  pivot/
    eval_types.py       # RolloutResult, EvalMetrics dataclasses
    reward.py           # ALP reward function
    eval_harness.py     # Evaluation metrics computation
    tokens.py           # Action token setup (<continue>, <refine>, <terminate>)
scripts/
  eval_pivot.py         # Evaluation CLI (YAML config support)
  generate_rollouts_pivot.py  # K-rollout generation via vLLM
  generate_sft_3action.py     # LLM-based SFT dataset construction
configs/
  eval_qwen_math_7b.yaml           # Eval config
  generate_rollouts_qwen25.yaml    # Rollout generation config
tests/
  test_pivot_reward.py   # ALP reward tests (17 tests)
  test_pivot_eval.py     # Eval harness tests (11 tests)
```

Old backoff-token code is preserved in `deprecated/`.

## Quick Start

```bash
# Run evaluation
python -m scripts.eval_pivot --config configs/eval_qwen_math_7b.yaml --n 10

# Run tests
python -m pytest tests/ -v
```
